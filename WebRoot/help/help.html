登陆配置---可使用IP范围

用于规定可以使用该系统的IP地址，如内容：192.168.5.1-192.168.5.250,则表示该范围内的IP可以使用该系统。

为空时默认为都可以使用该系统。

登陆配置---不可使用IP范围

用于限制不可使用该系统的IP地址，如：192.168.5.4,则表示该范围内的IP不可以使用该系统。

如果该IP既在可使用IP范围又在不可使用IP范围，则该IP无法使用该系统。

应用系统配置---允许再次配置的部门

该域值中的部门管理员可再将此资源自定义的权限分配给其他的用户、部门、职位、角色。

 

数据表---表单基本信息---记录标题

用于显示记录表单的标题，可自定义，如：导航栏配置――{F$navigation_name}，navigation_name为该表中的字段。

 

数据表---表单基本信息---显示HTML页面

用于自行配置当前页面显示的HTML，如果该项为空则使用系统默认HTML显示记录内容。

如内容：D:\Eclipse_WorkSpace\ucapPlatform\WebRoot\html\system\03D661949DBF98FC18EF4DF59F2044C3.html,则表示将使用该HTML页面显示记录内容。

数据表---表单基本信息---可编辑JS

用于判断页面是否可编辑，不符合条件时只读，如：1!=2，则表示该页面为只读页面。

为空时默认表示页面为可编辑页面。

_UcapForm.cfg.isRead：0为只读，1为编辑，-1为无权限

取父窗口父节点页面的文档状态（一般用于三级界面视图打开的文档）window.opener.parent._UcapForm.cfg.isRead

取父窗口的文档状态（一般用于二级界面视图打开的文档）window.opener._UcapForm.cfg.isRead

取当前页面的文档状态_UcapForm.cfg.isRead

以上_UcapForm.cfg.isRead可以替换成页面存在的js函数，如常用到的$V("字段名")可以从json中取到应的最初的字段数据

常用到的方法还有：$F("字段名")、$("字段名")等等

常用到的全局变量还有：ucapHeader.userJson.unid/depts/roles/effectiveFullDept等等

例如：$V("字段名")=="值"，返回true则为编辑，false则返回只读页面

if($V("字段名")=="值")_UcapForm.cfg.isRead="-1"//如果返回ture则返回无权限页面

数据表---表单基本信息---数据维护者

此用户可对此表单的所有相关文档进行修改，由系统记录相应的操作日志，包括对流程异常的处理。

为空时默认所有人都不可以进行修改。

数据表---字段基本信息---来源值

用于自行定义属性为字典、视图列、日期、日期时间、大文本、通用选择框字段的来源数据。

属性为字典、通用选择框时可通过选项来选择来源。

属性为日期：yyyy-MM-dd、yyyy/MM/dd等格式。

属性为日期时间：yyyy-MM-dd HH:mm:ss、yyyy/MM/dd HH:mm等格式。

数据表---字段基本信息---列配置对应

当字段属性为视图列时出现该配置项。

用来取得视图列的值，可配置视图中要取的列的字段名。

如:unid,title,name,age则表示取得字段名为这四个列的值。

数据表---字段显示属性---验证范围

用于字段内容长度的验证，为空时默认为无验证。

如：“100,600”表示长度范围在100-600间，“,100”表示小于100，“100,”表示大于100。注意：配置时不带引号。

数据表---字段显示属性---字段隐藏配置信息

隐藏与否特定条件:用于判断该字段是否隐藏，不符合条件的如果没有替换值，则该字段将隐藏。为空时默认为不隐藏。

如：1！=2，并且替换值为空则表示该字段隐藏。


隐藏替换值:与“隐藏与否特定条件”形成组合条件。用于替换该字段原有内容，不符合条件的字段内容将被替换。

如：1！=2，并且替换值不为空则表示该字段替换成“隐藏替换值中”的内容。

数据表---字段显示属性---字段编辑配置信息

编辑与否特定条件:用于判断该字段是否可编辑，不符合条件的为只读。默认为可编辑。

如：1！=2，则表示为只读。

数据表---按钮配置---显示与否JS

用于判断该按钮是否显示，不符合条件则该按钮不予显示,默认为显示。

如:1！=2则表示该按钮为隐藏不予显示。

数据表---按钮配置---所属菜单

指定按钮的父级的名称，应用时用菜单进行展示。

 

数据库视图配置---数据库视图条件

用于配置视图的WHERE显示条件，为空时默认无条件

如：where A.a1=B.b2则表示只显示符合这个条件的记录，不符合的记录将不予显示。

视图配置---视图基本信息---打开时js事件

单击该视图记录时触发的JS事件，为空时默认为打开该记录。

如:createRecord()则表示单击该视图时将执行这个JS函数。

视图配置---视图基本信息---增加行JS

在可编辑视图中，当用户在最后一行最后一列的单元格回车时执行该JS事件。

为空时默认为保存

视图配置---视图基本信息---视图加载完JS

该视图加载完之后触发的JS事件。

如：showWelcomeMessage()则表示视图加载完后将执行该JS函数。

二次开发---方式二（实现IAction接口）

前台

(1)前台采用EXJ2.0框架

(2)前后台交互的JS代码示例如下：
function doExecute(){
    var json = ucapCommonFun.getFormJSon("expDialogHtml");
    var requestConfig = {
      url:ucapSession.baseAction,
      params:{"type":"configLog","action":"excute"},
      jsonData:json,
      callback:function(options,success,response){
        if (success){
          var exjson = Ext.util.JSON.decode(response.responseText);
          if(exjson.result || exjson.result=="true"){
            Ext.Msg.alert("提示",exjson.msg);
            if(ucapSession.commonWin){
              ucapSession.commonWin.close();
            }
           }else{
             ; Ext.Msg.alert("提示",exjson.msg);
           }
          } else {
            Ext.Msg.alert("提示","执行不成功！");
            if(ucapSession.commonWin){
              ucapSession.commonWin.close();
            }
          }
        }
      }
          Ext.Ajax.request(requestConfig);
    }

说明：
（1）JSON---前台要传递给后台的内容值，值为ID为expDialogHtml里所有对象的值。
（2）"type":"configLog"---type为标识（固定为type），configLog为后台类的引用名。
与WebRoot\WEB-INF\verbs.properties相关联，说明前台提交到configLog在verbs.properties中所对应的后台类。
（3）"action":"excute"---action为参数名，excute为参数值，都可自行定义。
（4）exjson---后台返回的JSON结果值。
（5）在“按钮配置视图”里配置上拥有这个方法的按钮并将按钮应用到系统中，单击该按钮时就触发该方法。

JS包简单说明：
WebRoot\js\portal\form.js封装了关于表单的JS方法。
WebRoot\js\util\common.js封装了关于系统常用工具的JS方法，比如系统SESSION值等。
WebRoot\js\view\view.js封装了关于视图的JS方法

后台

(1)新建的类必须实现IAction接口。

(2)JSONObject requestStr = JsonUtil.readJsonFromRequest(request)---获取前台传过来的JSON值。

(3)JsonUtil.printJSONObject(response, JSONObject .fromObject("{result:true,msg:'导出成功'}"));---后台返回JSON值到前台。
